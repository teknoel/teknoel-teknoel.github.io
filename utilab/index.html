Numbler controls streamlines numerical input by forgoing pop-up keyboards and static up/down buttons.

Press briefly on a digit to increment; the top half increases value and the bottom decreases value. Drag vertically to scroll value.

The Numbler Class consists of a three-tiered object hierarchy which simplifies creating multiple digit input. Numblers ganged together can perform integer overflow. The number base system can be set to any size, even beyond hexadecimal, base 16. This is useful for input designs such as a clock in which the hours range from zero to 12 or 24 depending on the format. Customized digit scroll sequences can also be defined via a single callback routine.

SafeSlide is tailored for the execution of asynchronous processes. The linked process can provide detailed status of it's execution and update SafeSlides status text according to whether the process succeeded or failed. Three integrated text fields are available to provide feedback: Prompt, Status, and Value.

There are two modes of user engagement, both which relate to the behavior of the thumb component: Sticky and Commit. Sticky mode allows the thumb to self-lock in the engaged position. Commit Mode requires the linked process to a confirmation event in order for the thumb become locked in the engaged position.

Once a process has ended, there are two reset scenarios: Automatic or Single Shot. Automatic reset will restore the control to it's initial state, ready for immediate repeat usage. Single-shot disables the control after the process has ended. It is up to the application to re-enable the control.
